# OpenCode System Instructions

## Git Workflow
- Always checkout master before starting work
- Create feature/bugfix branches from master
- Never commit directly to master

## Commits
- Use Conventional Commits
- Allowed types: feat, fix, docs, style, refactor, test, chore

## Safety
- No destructive commands without confirmation
- Break large tasks into small verifiable steps

## Codebase Intelligence
- Use Graphify before exploring existing architecture or relationships when available
