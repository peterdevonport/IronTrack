### Description
The function creates a temp file via `mktemp` at line 76, but there is no `trap` to ensure cleanup on failure. With `set -euo pipefail` active (line 21), if either `curl` (line 77) or `bash` (line 78) exits with a non-zero status, the script terminates immediately and the `rm -f "$install_script"` on line 79 never executes.

While CI environments clean up workspace directories between runs, the temp file leaks on every failed install attempt. More importantly, this violates the defensive programming principle that resources acquired should be released in all code paths, not just the happy path.

### Suggested Refactor
Add a trap immediately after creating the temp file to ensure cleanup regardless of exit path:

```
local install_script
install_script=$(mktemp)
trap 'rm -f "$install_script"' EXIT
curl -fsSL https://opencode.ai/install -o "$install_script"
bash "$install_script" --version "${OPENCODE_VERSION}"
# No need for explicit rm -f — the EXIT trap handles it
```

Alternatively, use a subshell or a dedicated cleanup function to scope the trap to just this function rather than the entire script's EXIT handler.
