The PB badge and 1RM badge light-mode CSS overrides added by PR #382 are copy-pasted verbatim into two separate theme blocks:

- `.mdui-theme-light` (lines 97–105)
- `.mdui-theme-auto` (lines 137–145)

These 10 lines are character-for-character identical. This violates the DRY (Don't Repeat Yourself) principle: any future change to badge colors, opacity, or border values must be updated in both places. If a developer updates one block but misses the other, the two themes will silently diverge.

### Suggested Refactor
Extract the shared light-mode Tailwind class overrides (both slate and badge) into a single reusable block. Two strategies:

1. **CSS Nesting with shared class**: Define a shared class like `.mdui-light-overrides` containing all the badge + slate rules, and have both `.mdui-theme-light` and `.mdui-theme-auto` inherit it via `@extend` or class application in the HTML.

2. **CSS nesting grouping**: Since both selectors need the same overrides, use a combined selector:
   ```css
   .mdui-theme-light,
   .mdui-theme-auto {
     /* All shared light-mode Tailwind overrides here, once */
     .bg-purple-950\/50 { ... }
     .text-purple-400 { ... }
     /* etc. */
   }
   ```
   This eliminates the duplication while preserving the same cascade specificity.
