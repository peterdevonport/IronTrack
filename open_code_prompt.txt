### REFACTOR BLUEPRINT REPORTED:

### Location
Found in recent master commits.
- `theme.js:118-119`

### Description
In `initTheme()`, when the locally stored theme differs from the Firestore-saved theme, `persistTheme(mode)` is called without `await`. The `persistTheme` function is `async` and performs both a `localStorage.setItem` and a `setDoc` to Firestore. Without `await`, the Firestore write is fire-and-forget — any network failure or permission error is logged to console but the user gets no indication that their theme preference failed to sync to the server. On next page load, the stale Firestore value could overwrite the local choice.

### Suggested Refactor
Add error recovery: either `await` the `persistTheme()` call and handle the error in `initTheme`, or (better for UI responsiveness) make `persistTheme` return a Promise that is captured and handled with `.catch()`. Show a non-intrusive toast notification if the Firestore sync fails, and fall back to localStorage as the source of truth.
