### Description
PR #382 introduces six hardcoded color values that bypass the project's CSS custom property design token system:

- `rgba(168, 85, 247, 0.12)` — purple background
- `#7c3aed` — purple text
- `rgba(168, 85, 247, 0.3)` — purple border
- `rgba(16, 185, 129, 0.12)` — emerald background
- `#059669` — emerald text
- `rgba(16, 185, 129, 0.3)` — emerald border

The rest of the stylesheet follows a pattern of defining colors as CSS custom properties in `:root` and referencing them via `var()` (e.g., `var(--purple-400, #c084fc)` at line 415, `var(--color-brand)` at line 189). These new values are raw literals with no central definition, making them inconsistent with the established design system.

Furthermore, these badge light-mode CSS overrides are copy-pasted verbatim into two separate theme blocks:

- `.mdui-theme-light` (lines 97–105)
- `.mdui-theme-auto` (lines 137–145)

This violates the DRY (Don't Repeat Yourself) principle: any future change to badge colors, opacity, or border values must be updated in both places. If a developer updates one block but misses the other, the two themes will silently diverge. This is an extension of a pre-existing duplication problem — the slate Tailwind class overrides (lines 88–95 and 128–135) were already duplicated before this PR.

### Suggested Refactor
Two complementary refactors are recommended:

**1. Centralize badge color tokens:**
Define new CSS custom properties in the `:root` block for badge colors:

```css
:root {
  /* Badge tokens */
  --badge-purple-bg: rgba(168, 85, 247, 0.12);
  --badge-purple-text: #7c3aed;
  --badge-purple-border: rgba(168, 85, 247, 0.3);
  --badge-emerald-bg: rgba(16, 185, 129, 0.12);
  --badge-emerald-text: #059669;
  --badge-emerald-border: rgba(16, 185, 129, 0.3);
}
```

Then reference `var(--badge-purple-bg)` etc. in the light-mode overrides. This centralizes the values and aligns with the existing design token architecture.

**2. Eliminate duplicated theme blocks:**
Extract the shared light-mode Tailwind class overrides (both slate and badge) into a single reusable block using a combined selector:

```css
.mdui-theme-light,
.mdui-theme-auto {
  /* All shared light-mode Tailwind overrides here, once */
  .bg-purple-950\/50 { ... }
  .text-purple-400 { ... }
  /* etc. */
}
```

This eliminates the duplication while preserving the same cascade specificity.
